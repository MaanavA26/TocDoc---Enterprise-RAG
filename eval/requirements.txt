# RAGAS offline evaluation harness — STANDALONE dependency set (P4-2).
#
# This file is pinned INDEPENDENTLY of the services and is NOT a subset of
# any service's runtime image. The eval harness is a dev/CI tool only.
#
# WHY DECOUPLED (do not re-add `-r ../services/qna/requirements.txt`):
# The services now run the langchain 1.x cascade (langchain 1.3.2 family,
# langchain-core 1.4.0, langchain-community 0.4.2). ragas cannot yet run on
# langchain 1.x: ragas imports
#   langchain_community.chat_models.vertexai.ChatVertexAI
# at module load, and that symbol was removed in langchain-community 0.4.x.
# The langchain-community versions that still export it cap langchain-core<1,
# which is incompatible with the services. So the harness is pinned to the
# last langchain 0.3.x stack that ragas supports, fully self-contained here.
#
# The harness imports the QnA service source READ-ONLY (via sys.path, as
# `src.*`). QnA's only langchain imports are `langchain_openai`.AzureOpenAIEmbeddings
# and `langgraph.graph` (START/END/StateGraph) — both present in 0.3.x and 1.x —
# so the on-disk 1.x QnA source imports cleanly under this 0.3.x stack.
#
# Build a dedicated venv (python3.12) for this harness; do NOT install it into
# the QnA service venv:
#     python3.12 -m venv .venv-eval && . .venv-eval/bin/activate
#     pip install -r eval/requirements.txt
